# Laragod

Portfolio site for Laragod — a Laravel development agency. Also serves as the **reference implementation** for building new client sites using the [`laragod/toolkit`](https://github.com/laragod/toolkit) package.

## What this repo is

This is a real production site. It also demonstrates how to wire up `laragod/toolkit` in a Laravel project. When starting a new client site, use this repo as a reference — not a fork.

## Stack

- PHP 8.4, Laravel 12
- Blade + Tailwind CSS 4.0
- PHPUnit 11
- [`laragod/toolkit`](https://packagist.org/packages/laragod/toolkit) — shared backend infrastructure

## Local setup

```bash
git clone https://github.com/laragod/laragod.git
cd laragod
composer install
cp .env.example .env
php artisan key:generate
npm install && npm run build
php artisan test
```

## Starting a new client project

Install a fresh Laravel app and require the toolkit:

```bash
composer create-project laravel/laravel client-name
cd client-name
composer require laragod/toolkit
php artisan vendor:publish --tag=laragod-toolkit
```

Then wire up three things, using this repo as reference:

### 1. `bootstrap/app.php` — register `SetLocale` middleware

```php
use Laragod\Toolkit\Http\Middleware\SetLocale;

->withMiddleware(function (Middleware $middleware): void {
    $middleware->web(prepend: [
        SetLocale::class,
    ]);
})
```

### 2. `routes/web.php` — hook up toolkit controllers

```php
use Laragod\Toolkit\Http\Controllers\HealthController;
use Laragod\Toolkit\Http\Controllers\SitemapController;
use Laragod\Toolkit\Http\Middleware\RedirectToLocale;

// Health checks
Route::get('ping',   [HealthController::class, 'ping']);
Route::get('health', [HealthController::class, 'healthCheck']);
Route::get('status', [HealthController::class, 'status']);

// Sitemap
Route::get('sitemap.xml', [SitemapController::class, 'index'])->name('sitemap');

// Root → locale redirect
Route::get('/')->middleware(RedirectToLocale::class);

// Localized pages
Route::prefix('{locale}')
    ->where(['locale' => implode('|', array_keys(available_locales()))])
    ->group(function (): void {
        // your controllers here
    });

// Contact POST (CSRF-protected, rate-limited)
Route::post('/contact', [ContactController::class, 'store'])
    ->middleware('throttle:5,1')
    ->name('contact.store');
```

### 3. `app/Http/Controllers/ContactController.php` — inject `NotificationManager`

```php
use Laragod\Toolkit\Services\NotificationManager;

class ContactController extends Controller
{
    public function __construct(
        private readonly NotificationManager $notifications,
    ) {}

    public function store(Request $request): JsonResponse
    {
        $data = $request->validate([...]);
        $sent = $this->notifications->sendContactNotification(...);
        return $sent
            ? response()->json(['ok' => true])
            : response()->json(['ok' => false], 500);
    }
}
```

### 4. `config/sitemap.php` — list your controllers

```php
return [
    'base_url'    => env('SITEMAP_BASE_URL', 'https://yourclientsite.com'),
    'controllers' => [
        App\Http\Controllers\FrontController::class,
        App\Http\Controllers\ContactController::class,
    ],
];
```

### 5. Build your views

The toolkit provides no views. Build `resources/views/` from scratch for the client's design.
